IntraWeb XI has a new pallete with components for user authentication. Currently there are 3 components: TIWAutherINI, TIWAutherList and TIWAutherEvent. The ServerController has the new property Auther, which must be used to connect it to any of the available authentication components.

Authentication Process

If no auther exists, no authentication will ever be requested of the user.

If an auther exists:

  • The auther will be responsible for validating the username and password supplied when authentication is requested.
  • TIWAuther.Policy controls the default mechanism for requests. Policy can be used to control whether requests required authentication or not by default.
  • This default can be overriden on a form by form basis by implementing a RequestAuth method. If not override exists for a form, the default set by Policy is used.
